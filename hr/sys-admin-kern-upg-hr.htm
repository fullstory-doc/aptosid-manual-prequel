Instalirati novi kernel

The sidix kernels reside here and are available in the following forms:

  • up - Kernel for the i686 Processor Family
  • smp - Kernel for the i686 Processor Family with dual core or more CPUs
  • 64-smp - Kernel for the AMD64- Processor Family, when using 64 bit sidux

Note down the kernel name you require for your processor (we call this archive kernel.zip, but its going to have a name like: kernel-2.6.20.4-slh-up-1.zip ), make a note of it then:

To install as user
Its 5 lines to do it manually in a konsole:
mkdir /home/username/kernel-downloads/
cd ~/kernel-downloads
wget -Nc http://sidux.com/files/kernel/kernel-2.6.20.4-slh-up-1.zip
unzip kernel-2.6.20.4-slh-smp-1.zip -d kernel-2.6.20.4-rc1-slh-smp-1
cd kernel-2.6.20.4-rc1-slh-smp-1
./install-kernel-sidux.sh (it will ask for your root password)
To install as root
sux
cd /usr/src
wget -Nc http://sidux.com/files/kernel/kernel-2.6.20.4-slh-up-1.zip
unzip kernel-2.6.20.4-slh-up-1.zip  -d kernel-current
cd /usr/src/kernel-current
sh install-kernel-sidux.sh

Now restart your computer. The new kernel will boot up by default. Should the new kernel give problems, you can reboot and choose an older kernel try this stage again

IMPORTANT: if you need certain driver modules, e.g. for wlan,you have to install the proper .deb packages for them, e.g., if using madwifi, you have to install the .deb package for madwifi: # dpkg -i madwifi-modules*.deb

The kernel-update is now done. Should you require them, add the latest graphics driver which can be installed by referring to 3D drivers, If you have an ATI or NVIDIA card, the first reboot to the new kernel should be made to runlevel 3 (this is done by adding 3 to the Grub-Boot-Line)

Removing old kernels

After successfull installation of the new kernel, the kernel.zip (and the kernel archive) can be deleted, however its recommended that you keep them for a few days in case you strike problems and therefore boot to an older kernel as listed in the grub screen

Old kernels can be removed from the system. To do so, you can use the script kernel-remover which is in the package sidux-kernelhacking! This script allows to you nominate which kernels you wish to remove.
